## What this is

Rapoport Studio is a Moldovan SRL building digital platforms with AI as the
orchestration substrate. This repo carries the studio's methodology corpus —
the conventions, charters, and roadmap that shape every engagement.

It is the public reference behind:

- The trust pillar on [rapoport.studio](https://rapoport.studio) — *"Spec is markdown in your repo. From day one. Every commit is yours."*
- The GitHub App **Rapoport Studio Connect** description.
- The engagement charter that prospective clients can read before signing.

The studio's source repo (`rapoport-studio/rapoport.studio`) stays private.
This mirror carries the _methodology_ surface only — the parts that describe
*how the studio operates*, not *what the studio is currently building*.

## What's intentionally NOT here

- `current/` — internal capability specs. Architecture detail, not useful publicly.
- `changes/` — in-flight proposals. Would expose unfinished thinking.
- `archive/` — the studio's audit log. Useful internally, noise externally.
- `mirror/` — per-engagement Mirror perception files. NDA-bound.
- `_methodology/projects/` — project-specific prompts and YAML profiles. Reference clients privately.

If you want deeper access to engagement-specific artifacts, that comes through
your engagement (charter §§ 4–5).
