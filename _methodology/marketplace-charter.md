# Marketplace charter

> One-paragraph charter. Detailed design deferred.

The Marketplace is a horizontal peer-to-peer transaction layer where platform users sell each other templates, re-usable specs, ideas, and knowledge across engagements. The Marketplace owns transactions — listing, discovery, pricing, fulfilment, fees; what is transacted lives in the existing three spheres — a template is a Spec artifact, a brand pattern is a Brand artifact, an idea is an entity in the platform's idea space. The Marketplace is not part of the studio's vertical offer (`Spec + Brand + Forge`); it is orthogonal to it. The studio operates the substrate; users produce the inventory.

Detail — folder shape, entity schema, listing flow, pricing model, fee structure, dispute handling, listing surfaces — is **scope-deferred**. This stub exists so the concept has a citable home for cross-references in other documents. Work on the Marketplace is gated on (a) the studio's Spec / Brand / Forge pillars reaching maturity, and (b) pilot evidence that real users want to transact, not just consume.
